    Introduction

    Below documentation highlights the parameters as needed for each of the API.

    Authentication

    Authentication key and user ID is needed for access to the APIs. To get the key and the user ID, email us at hello@liv.ai.

    The API key has to be included in the header of all Https requests to the server.

    Authorization: Token API_key

    All POST requests should also include the user ID in the body of the request.

    user = user_ID

    Replace API_key with your API key and user_ID with your user ID.
    Note the space between ‘Token’ and the API key.

    Tags Upload API

    curl \
  -H "Authorization: Token exampleapikey1" \
  --form "user=123" \
  --form "language=EN" \
  --form "tag_file=@/path/tags.txt" \
  https://dev.liv.ai/liv_speech_api/tags/

    import requests

headers = {'Authorization' : 'Token exampleapikey1'}
data = {'user' : '123', 'language' : 'EN'}
files = {'tag_file' : open('tags.txt','rb')}
url = 'https://dev.liv.ai/liv_speech_api/tags/'
res = requests.post(url, headers = headers, data = data, files = files)

    The above code returns a JSON structured like

    {
  "status" : "success"
}

    Use this API to upload tags.

    Https URL

    https://dev.liv.ai/liv_speech_api/tags/

    Https Method

    POST

    Header

    Authorization: Token API_key

    Request Body

    Field Description
    tag_file Text or csv file with a list of tags to be uploaded.
    File should contain one tag per line.
    user Your User ID.
    language Language code of the tags
    Language Code
    English EN
    Hindi HI

    Response

    Field Description
    status Status with “success” message.
    File size should be less than 500 kB.
    In case of error, refer detail key for error details.

  -H "Authorization: Token exampleapikey1" \
  --form "user=123" \
  --form "language=EN" \
  --form "audio_file=@/path/test.m4a" \

    import requests

headers = {'Authorization' : 'Token exampleapikey1'}
data = {'user' : '123' ,'language' : 'EN'}

res = requests.post(url, headers = headers, data = data, files = files)

    The above code returns a JSON structured like

    {
  "app_session_id": "exampleappsessionid1"
}

    Use this API to upload single audio file per session.
    Accepted file types:

    • m4a
    • mp3
    • aac

    Https URL

    https://dev.liv.ai/liv_speech_api/recordings/

    Https Method

    POST

    Header

    Authorization : Token API_key

    Request Body

    Field Description
    user Your user ID.
    audio_file The file to be uploaded.
    language Language code of the audio file.
    meta Auxiliary data (OPTIONAL).
    Language Code
    English EN
    Hindi HI

    Response

    Field Description
    app_session_id Session ID of the session.
    Use multipart/form-data for the body of the request.
    File size should be less than 10 MB.
    In case of error, refer detail key for error details.

    Status API

    curl \
  -H "Authorization: Token exampleapikey1" \
  -X GET \
  https://dev.liv.ai/liv_speech_api/session/status/?app_session_id=exampleappsessionid1


    import requests

headers = {'Authorization' : 'Token exampleapikey1'}
params = {'app_session_id' : 'exampleappsessionid1'}
url = 'https://dev.liv.ai/liv_speech_api/session/status/'
res = requests.post(url, headers = headers, params = params)

    The above code returns a JSON structured like

    {
  "upload_status": 20,
  "transcribed_status": false,
  "tags_status": true
}

    Use this API to check session status.

    Https URL

    https://dev.liv.ai/liv_speech_api/session/status/

    Https Method

    GET

    Header

    Authorization : Token API_key

    Parameters

    Parameter Description
    app_session_id The app session id of the session for which you are checking status.

    Response

    Field Description
    upload_status Upload status code.
    transcribed_status Status of transcription (True/False).
    tags_status Status of tags (True/False).
    Upload Status Code Details
    0 Unprocessed session.
    10 Incomplete session where last part is received but some parts are missing.
    11 Incomplete session where last part is not received.
    20 Complete session. Session closed and no more files can be added.
    -10 Session immaturely closed due to insufficient number of uploads.
    In case of error, refer detail key for error details.

    Session Tags API

    curl \
  -H "Authorization: Token exampleapikey1" \
  -X GET \
  https://dev.liv.ai/liv_speech_api/session/tags/?app_session_id=exampleappsessionid1


    import requests

headers = {'Authorization' : 'Token exampleapikey1'}
params = {'app_session_id' : 'exampleappsessionid1'}
url = 'https://dev.liv.ai/liv_speech_api/session/tags/'
res = requests.post(url, headers = headers, params = params)

    The above code returns a JSON structured like

    {
  "app_session_id": "exampleappsessionid1",
  "tags": [
    {
      "confidence_score":0.866539857262059,
      "text_hindi":"आफ्टरनून",
      "text_eng":"afternoon",
      "time_sec":13.16
    },
    {
      "confidence_score":0.91951566175981,
      "text_hindi":"डोनट",
      "text_eng":"don't",
      "time_sec":5.06
    }
  ]
}

    Use this API to get the tags associated with a particular session.

    Https URL

    https://dev.liv.ai/liv_speech_api/session/tags/

    Https Method

    GET

    Header

    Authorization : Token API_key

    Parameters

    Parameter Description
    app_session_id The app session id of the session for which tags are required.

    Response

    Field Description
    app_session_id Session ID of the session.
    tags List of tags associated with the session.

    The fields in each tag are explained below.

    Field Description
    confidence_score Confidence level of extracted phrase.
    text_hindi Phrase in Hindi.
    text_eng Phrase in English.
    time_sec Time of occurrence of phrase in the audio (in seconds).
    In case of error, refer detail key for error details.
